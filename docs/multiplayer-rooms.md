# 훌라 멀티모드 — 다중 방(room) 시스템 기획

## 배경 (현재 구조, 코드 확인 완료)

- `waitingRoom`(전역 Map 1개), `activeGame`(전역 변수 1개) — 서버 전체에 게임이 딱 1개만 존재 가능
- 유저는 admin이 미리 등록해둔 최대 20명 (`isAdmin` 플래그 포함)
- `sendInvite`: isAdmin만 가능
- `adminStartGame`: isAdmin만 가능
- `joinMulti`: 사이트 전체 입장코드(entryCode)로 게이트, 이건 유지

## 목표

1. 동시에 여러 그룹이 각자 독립적으로 게임 가능 (다중 방)
2. 초대/게임시작을 admin 아닌 누구나 가능하게
3. 초대 대상은 여전히 기존 등록 유저(최대 20명) 안에서만
4. 방 코드로도 입장 가능 (직접 초대 + 코드 입장 둘 다 지원)

## 아키텍처 변화 (개념)

```
현재                              바뀔 모습
waitingRoom (전역 1개)      →      rooms: Map<roomCode, RoomState>
activeGame (변수 1개)       →      각 방마다 자기 게임 상태
```

- 방마다: 제목, 코드, 참가자 목록, 게임 상태
- 세션(session)에 "지금 어느 방에 있는지" 추가 필요
- AI로 4명 채우는 로직(aiPlayer.js)은 방 단위로 재사용

## 전역 흐름: 받은 초대 알림

앱 어디에 있든(메인, 싱글게임 중, 멀티 로비 등) 초대가 오면 알림이 뜸.
- 수락: 하던 것(싱글게임 등)은 일시정지(`pausedReason: 'invite'`, 기존 로직 재사용) → 바로 해당 방 대기실로 이동
- 거절: 알림만 닫힘, 하던 일 계속

## 멀티 로비 화면

```
멀티모드 진입 (기존 입장코드 체크 유지)
  → 멀티 로비
     - 방 목록 (제목, 인원수 N/4)
     - [방 만들기]
     - [방 코드로 입장]
```

## 방 만들기 (수동)

- 방 제목: 사용자가 직접 입력
- 방 코드: 사용자가 직접 입력 (중복이면 에러, 재입력 요청)
- 초대할 사람 선택 (선택 사항, 건너뛰기 가능)
  - 목록 대상: 현재 온라인이면서 "싱글게임 중"이거나 "아무 방에도 없는" 사람만 (다른 방/멀티게임 중인 사람 제외)
  - 최대 3명 체크 (본인 포함 4명 정원)
- 완료 시: 방 생성, 체크한 사람에게 초대 전송 (싱글게임 중이었다면 일시정지 후 초대)

## 방 코드로 입장

- 로비에서 코드 직접 입력 → 해당 방 존재하고 정원 남아있으면 입장

## 랭킹 화면에서 빠른 초대 (신규)

- 랭킹 화면에 [초대] 버튼 추가
- 누르면 순위(1위/2위...) 표시 자리가 체크박스로 토글됨
  - 온라인인 사람만 체크 가능, 최대 3명
  - 오프라인인 사람은 체크 비활성(회색)
- [초대 보내기] 누르면:
  - 방 자동 생성 (제목: "{방 만든 사람 이름}의 방", 코드: 내부 자동생성이나 초대받은 사람은 볼 필요 없음)
  - 체크한 사람들에게 일괄 초대 전송
  - 초대 보낸 사람은 바로 해당 방 대기실로 이동
  - 초대받은 사람은 수락 시 코드 없이 바로 해당 방 대기실로 입장

## 방 대기실

- 참가자 목록 표시
- 누구나 다른 사람 초대 가능 (대상 제약은 위와 동일)
- 2명 이상 모이면 누구나 "게임 시작" 가능 (AI가 나머지 자리를 채워 4명)

## 열린 질문 (아직 미정)

- 방 목록에 "싱글게임 중이거나 아무 방에도 없는 사람" 필터가 방 생성 시 초대 대상 목록에만 적용되는지, 방 목록 자체의 노출 범위(예: 이미 진행 중인 게임방도 목록에 보이는지, 참가 불가 표시만 하는지)는 아직 안 정함
- 방이 비면(모두 나가면) 자동 삭제되는지, 게임 종료 후 방은 어떻게 되는지(재시작 가능한지, 사라지는지) 미정
