5. 턴 구조
----------------------------------------------------------------

[1단계: 드로우 (필수)]
- 카드 더미 맨 위 1장
- 버린 더미 맨 위 1장 (첫 턴 제외)
- 둘 중 하나 선택

[2단계: 행동 (선택, 여러 번 가능, 순서 무관)]
- 등록: 카드 여러 장 선택 후 등록 버튼 클릭 (한 번에 하나의 조합)
- 붙이기: 최초 등록 완료 후에만 가능
          카드 1장 선택 → 붙일 조합 클릭 → 시스템이 자동 위치 배정
          본인 또는 타인의 등록된 조합 어디에나 카드 추가 가능
          이미 등록/붙인 카드는 떼서 이동 불가
- 카드 선택 취소: 선택한 카드 다시 클릭 시 선택 해제
- 유효하지 않은 조합/붙이기: "유효하지 않은 조합입니다" 메시지

[3단계: 버리기 (필수)]
- 1장 선택하여 버림
- 버리는 즉시 턴 종료

[타이머: 60초]
- 드로우 안 한 상태: 카드 더미 / 버린 더미 중 랜덤 드로우 → 최솟값 카드 자동 버림
- 드로우는 했지만 버리기 안 한 상태: 최솟값 카드 자동 버림
- 땡큐 카드를 가져간 상태에서 60초 초과: 자동 취소 + 벌금 처리 (수동 취소와 동일)
  → 시간 초과 안내 모달 표시

[즉시 승리]
- 턴 중 언제든 손패 0장이 되는 순간 즉시 승리

7. 등록 조합 규칙
----------------------------------------------------------------
- 같은 숫자 3장 이상 (무늬 무관)                    → 세트
- 같은 무늬 + 연속 숫자 3장 이상                    → 시퀀스
  순환 허용: A-2-3, K-A-2, Q-K-A 모두 가능 (방향 제한 없음)
- 7 두 장 (서로 다른 무늬) 등록 가능               → 세트 방향
- 7 한 장 단독 등록 가능                            → 세븐
- 7 단독 등록 후 붙이기:
    같은 숫자(7) 추가 시        → 세트 방향으로 발전
    같은 무늬 인접 숫자 추가 시 → 시퀀스 방향으로 발전
    한 번 방향 결정되면 해당 규칙 유지
- 7이 포함된 시퀀스는 2장부터 가능 (예: ♠6-♠7, ♠7-♠8)
  7이 없는 시퀀스는 반드시 3장 이상
- 등록 확정 후 해제 불가

9. 땡큐 시스템
----------------------------------------------------------------

[발동 순서]
1. A가 카드를 버림 → 즉시 B(다음 차례)의 턴이 됨
2. B가 드로우하기 전까지 누구나 (A 제외) 땡큐 버튼 클릭 가능
3. B가 카드 더미에서 드로우하는 순간 땡큐 창 자동 비활성화

[누가 땡큐 가능한가]
- 카드를 버린 사람(A) 제외한 모든 플레이어 가능
- 다음 차례인 플레이어(B)도 땡큐 가능
  → B가 드로우 대신 땡큐를 선택하면 B의 행동 단계로 진입

[서버 처리]
- 선착순 1명만 성공 (LOCK 방식, 서버 수신 시간 기준)
- 나머지 요청은 자동 무효

[땡큐 성공 시]
- 해당 카드 획득 → 행동 단계부터 시작
- 땡큐로 가져온 카드로 행동(등록/붙이기) 먼저 해야 함
- 해당 행동 완료 후 추가 행동 가능
- 다음 턴 순서: 땡큐 성공자 기준 시계 방향으로 계속
  예) A→B→C→D에서 A 버림, C 땡큐 성공 → C 행동 → D→A→B→C 순

[땡큐 취소]
- 땡큐로 카드를 가져온 후 "취소" 버튼 클릭 가능
- 취소 시: 가져온 카드 버린 더미 맨 위로 반환
         원래 다음 차례였던 플레이어(B) 턴으로 복귀 (draw 단계)
- 취소 후 본인이 다시 땡큐 가능
- 취소 버튼 자동 사라지는 조건:
    가져온 카드를 등록에 사용했을 때
    가져온 카드를 붙이기에 사용했을 때
- 벌금: 다른 플레이어 수 × 100원(멀티) / 1점(싱글)
  예) 4인 게임 → 취소자 -300원, 나머지 각 +100원

[버튼 상태]
- 활성화: 카드 버려진 직후 클릭 가능
- 클릭 시: "땡큐!" 표시 후 서버 판정 대기
- 성공: 카드 내 패로 이동
- 실패: "다른 사람이 가져갔어요" 표시
- 비활성화 상태에서 클릭: "이미 선점됐어요" 표시

[AI 땡큐 딜레이]
- 유용한 카드: 5~7초 후 높은 확률로 땡큐
- 유용하지 않은 카드: 6~8초 후 낮은 확률로 땡큐
- AI는 자신이 버린 카드, 자신이 현재 차례일 때는 땡큐 안 함
  (AI 현재 차례일 경우 덱에서 드로우하는 방식으로 처리)

10. 게임 종료 조건
----------------------------------------------------------------

[즉시 승리]
- 자신의 패가 0장이 되는 순간 승리
  (등록/붙이기로 0장, 또는 버리기 후 0장 모두 해당)
- 마지막 카드를 버릴 때: 빈 패를 먼저 보여준 뒤 승리 화면 표시

[카드 더미 소진 시]
- 마지막 카드 더미를 가져간 사람이 버리는 행동까지 완료되면 게임 종료
- 해당 버리기 후 땡큐 창 없이 즉시 종료
- 버린 더미 재사용 없음
- 현재 패 합산으로 순위 결정
  1순위: 카드합 낮은 사람
  2순위: 카드 장수 적은 사람
  완전 동점: 공동 순위 처리

- 아니면 카드 더미 선택
- 땡큐 창 활성 중(자신이 현재 플레이어일 때): 무조건 카드 더미에서 드로우
- 버린 더미가 hidden 상태(방금 드로우되어 뒤집힌 직후)에도 덱에서만 드로우

[등록/붙이기]
- 유효한 조합이 손패에 있으면 반드시 등록
- 붙일 수 있으면 붙이기

[버리기]
- 조합에 쓸모없는 카드 중 가장 높은 점수 카드 버리기

[행동 딜레이]
- 일반 AI 턴 시작: 약 1.5~2.5초 딜레이 후 시작
- 싱글모드 첫 번째 AI 턴: 5~8초 딜레이 (초보자 여유 시간 확보)
- 멀티모드 게임 시작/한판더 후 첫 AI 턴: 2~3초 딜레이
- 땡큐 성공 후 AI 행동: 2~3초 딜레이

[땡큐 판단]
- 카드 버려진 후 5~7초(유용) / 6~8초(불필요) 뒤 판단
- 유용한 카드: 70% 확률로 땡큐
- 유용하지 않은 카드: 10% 확률로 땡큐
- 자신이 현재 차례이거나 자신이 버린 카드면 땡큐 안 함
- 7은 항상 유용한 카드로 판단 (단독 등록 가능하기 때문)
- 취소 후 재땡큐 기회 발생 시: 유용 60% / 불필요 10%, 3~7초 딜레이로 재시도

14. UI 구성
----------------------------------------------------------------

[앱 진입 플로우]
앱 실행 → userCode 입력 (쿠키 자동로그인)
→ 메인 화면
   ├ 싱글모드 → 바로 게임 시작
   └ 멀티모드 → 입장 코드 입력 (최대 5회) → 대기 → 관리자 시작

[메인 화면]
- 싱글모드 버튼 (현재 포인트 표시)
- 멀티모드 버튼 (현재 잔액 표시)
- 랭킹 버튼
- 설정 버튼
  일반 유저: 승리 문구 설정 / 캐릭터 아이콘 선택
  관리자 (비밀번호 입력 후): 입장 코드 변경, 유저 관리 추가
- 도움말 버튼 (게임 규칙 + 사용법)

[사용자 캐릭터 아이콘]
- 기본(디폴트): 사람 아이콘 (픽셀아트)
- 설정에서 변경 가능
- 선택 가능 동물: 고양이, 곰, 토끼, 여우, 개구리,
                  판다, 코알라, 사자, 고슴도치, 늑대, 너구리, 소

[승리 문구]
- 기본값: "오예!" / 최대 20자 / 설정 화면에서 변경

[게임 테이블 화면 레이아웃]
모바일 가로(landscape) 기준, 클릭 방식

            [ 상대방 (위) ]
[ 상대방(좌) ]  [ 등록 조합 | 카드더미 | 버린더미 + 땡큐버튼 ]  [ 상대방(우) ]
  [ 내캐릭터 + 보유금액 + 등록여부 | ⊙60초 카운트다운 | 내패 + 액션버튼 ]

- 상대방 영역: 이름 + 현재 턴 강조(은은한 깜빡임) + 보유금액/등록여부 표시
- 상대방 패: 뒷면 표시 (장수 별도 미표시)
- 내 카드 선택: 클릭 시 카드 위로 올라오는 시각화 (다시 클릭 시 선택 해제)
- 내 패: 겹쳐서 가로 나열, 정렬 버튼(숫자순/무늬순/원래순서)
- 60초 카운트다운: 원형 안에 숫자 감소, 15초 이하 시 강조색
- 현재 턴 표시: 해당 플레이어 캐릭터 영역 + 이름 강조
- 땡큐 버튼: 버린 더미 옆, 발동 조건 충족 시 활성화
  비활성 조건: 자신이 버린 카드 / 이미 누가 눌렀을 때
- 관리자 전용: 게임 시작/중단 버튼
- 게임 로그: 모든 이벤트 자동 기록 (관리자 포함)

[등록 조합 표시]
- 겹쳐서 표시, 가장 오른쪽 카드가 완전히 보임

[승리 연출]
- 1등 영역 주변 꽃가루 효과
- 유저 설정 승리 문구 표시
- 확인 버튼 클릭 시 정산 화면으로 이동

[정산 화면]
- 컬럼: 순위 | 이름 | 카드점수 | 이번 게임 손익 | 총 보유 금액 | 총 승리 횟수
- 버튼: [한판 더] / [메인으로]

[랭킹 화면]
- 멀티 / 싱글 탭 분리
- 멀티 컬럼: 순위 | username | 총 보유 금액 | 총 승리 횟수 | 총 게임 수
- 싱글 컬럼: 순위 | username | 포인트 | 총 승리 횟수 | 총 게임 수
- 정렬: 보유금액/포인트 내림차순 → 승리 횟수 내림차순
- 전체 유저 표시 (최대 20명)

[기타 UI]
- 땡큐 취소 시: 손익 문구 플로팅 표시 (예: -300, +100)
- 채팅 없음 / 사운드 없음
- 게임 시작 시 카드 배분 애니메이션
- 땡큐 취소 버튼 클릭 시 확인 모달 표시 (벌금 안내 + 확인/닫기)
- 60초 초과 자동 취소 시 안내 모달 표시 (확인 버튼)
- 대기실 화면에 나가기 버튼 (leaveWaiting)

----------------------------------------------------------------

- 기본 관리자: userCode=kkkyyy123, userName=관리자, isAdmin=1
- 기본 입장 코드: 1234
- 기본 관리자 비밀번호: dd291234

16. 핵심 버그 수정 이력 (구현 중 확정된 사항)
----------------------------------------------------------------

[땡큐 창 관련]
- 자신이 버린 카드는 본인이 땡큐 불가 (discarderCode 서버/클라이언트 양측 차단)
- 현재 차례 플레이어(B)도 땡큐 가능 (버린 사람만 제외)
- 드로우 중 덱/버린더미 클릭 안 되던 버그: thankYou 상태 즉시 해제 후 nextTurn 호출로 수정
- 버린 더미 드로우: 땡큐 창 활성 중에는 버린더미 드로우 차단 (덱 드로우만 허용)
- 땡큐 취소 후 원래 차례 플레이어(B)로 복귀 (confirmThankYou 시 displaced 인덱스 저장)
- 땡큐 가져온 카드를 등록/붙이기에 사용하면 취소 버튼 즉시 사라짐

[턴 진행]
- 카드 버린 즉시 다음 차례로 전환 + 땡큐 창 동시 활성
- 마지막 덱 카드를 가져간 사람이 버리면 땡큐 창 없이 즉시 게임 종료 (lastDeckDraw 플래그)

[타이머]
- 땡큐 창이 열려있어도 내 차례면 60초 카운트다운 정상 시작

[승리 연출]
- 마지막 카드 버릴 때 빈 패 먼저 표시 → 0.7초 후 승리 화면

[패 렌더링]
- 애니메이션 중인 카드(.card-leaving)를 패 비교에서 제외 → 불필요한 전체 재렌더 방지
- 카드 선택 시 위로 올라오는 시각화가 3장 이상에서 잘리던 버그:
  #my-hand padding-top 추가 + selected 카드 z-index 설정으로 수정

[포인트 표시]
- 게임 중 내 싱글 포인트/멀티 잔액이 0으로 표시되던 버그:
  getPublicState에 singlePoints/multiBalance 포함

[멀티 재시작]
- "한판더" 시 대기실 거치지 않고 이전 게임 동일 멤버로 재시작
- 관리자가 중복으로 생성되던 버그 해소

[7 두 장]
- 7 두 장(서로 다른 무늬)을 선택해서 한 번에 등록 가능 (세트 조합)
- AI도 7 쌍을 유효 조합으로 인식
